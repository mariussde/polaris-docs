## Projektstruktur-Übersicht

### app

Dieser Ordner enthält den Hauptanwendungscode für Next.js und verwaltet Layouts, Routing und spezifische Inhaltsseiten. Er ist organisiert, um sowohl die `docs`- als auch die `blog`-Bereiche zu unterstützen, mit dedizierten Seiten und Layouts für jeden Bereich.

<Files
  items={[
    {
      type: "folder",
      name: "app",
      isOpen: true,
      children: [
        { type: "file", name: "page.tsx" },
        { type: "file", name: "layout.tsx" },
        {
          type: "folder",
          name: "blog",
          children: [
            { type: "file", name: "page.tsx" },
            {
              type: "folder",
              name: "[slug]",

            },

        {
          type: "folder",
          name: "docs",
          children: [
            { type: "file", name: "layout.tsx" },
            { type: "file", name: "[[...slug]].tsx" },
          ],
        },
      ],
    },
  ]}
/>

### lib

Dieser Ordner enthält Hilfsfunktionen und Utilities, die in der gesamten Anwendung verwendet werden, wie Markdown-Parsing und Routing-Logik. Diese Utilities helfen dabei, die Codebasis sauber und organisiert zu halten, indem sie gemeinsame Funktionalität auslagern.

<Files
  items={[
    {
      type: "folder",
      name: "lib",
      isOpen: true,
      children: [
        { type: "file", name: "markdown.ts" },
        { type: "file", name: "routes-config.ts" },
        { type: "file", name: "utils.tsx" },
      ],
    },
  ]}
/>

### Zusätzlich

- **`package.json`**: Enthält Metadaten über das Projekt, Abhängigkeiten und Skripte zum Erstellen und Ausführen der Anwendung.
- **`tailwind.config.ts`**: Konfiguriert Tailwind CSS und ermöglicht die Anpassung von Theme-Farben, Schriftarten und responsiven Breakpoints, die spezifisch für dieses Projekt sind.

Diese Struktur organisiert Ihr Projekt so, dass sie Skalierbarkeit und Wartbarkeit unterstützt und das Navigieren und Arbeiten an verschiedenen Bereichen der Anwendung erleichtert.
